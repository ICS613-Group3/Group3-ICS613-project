# **1\. Team Purpose**

*The team exists to practice rigorous collaborative software engineering methodologies, applying iteration, automated testing, and agile workflows to solve local community challenges. The project will deliver a secure web application enabling localized, high-trust peer-to-peer asset maximization.* 

# **2\. Project Scope and Objectives**

## **Project Vision**

*A highly responsive web interface enabling trust-verified neighbors to optimize resource sharing, coordinate pickups, and track tool-lending histories securely.*

## **Goals**

*Add 3–5 measurable or concrete project goals.*

* **Responsive Product Delivery:** Develop and deploy a responsive web application for invite-only neighborhood tool sharing that is feature-complete and functionally validated by the deadline.   
* **Comprehensive Reservation Lifecycle:** Architect and implement the full reservation state machine—navigating through **REQUESTED**, **APPROVED or DENNIED, CANCELLED**,  **PICKED\_UP**, and **RETURNED**—while strictly enforcing business rules to prevent double-booking.  
* **Quality Assurance & Robustness:** Validate system integrity through comprehensive automated and manual testing, specifically targeting edge cases and negative-path scenarios such as invalid date ranges or unauthorized permission attempts.

* **Professional Version Control & Collaboration:** Leverage the full GitHub ecosystem—including branch-based development, pull requests for peer code reviews, and milestone tracking—to manage the software development lifecycle. 

* **Iterative Agile Development:** Adopt an iterative workflow aligned with course milestones, maintaining a backlog of 25–30 detailed user stories to ensure transparency and steady incremental progress .

## **In Scope**

*List the features, responsibilities, or deliverables the team is committing to.*

* Invitation to join to access-controlled community  
* Account creation  
* Building a user profile

* Listing tools

* Searching tools

* Requesting a book

* Status changes (REQUESTED, APPROVED, PICKED\_UP, CANCELLED,  RETURNED or DENIED)

* Double-booking prevention

* Private messages

* Notification for status change

* Review/rating

* Having member dashboard

* Creating an admin role

* Admin can create reports, disable listings, and inactive users

## **Out of Scope**

*List major items the team is intentionally not taking on.*

* Discussion functionality among all members  
* Mobile apps  
* Each non-admin member suspend other members
