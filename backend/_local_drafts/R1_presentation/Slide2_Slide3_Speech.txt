Slide 2 Speech — Domain Model & ER Diagram (60 seconds)
=========================================================

Here is our domain model.

Three core parts: Users, Reservations and tools.

Users register by admin invite. Account states are email
pending, active, suspended, or deleted.

Members list tools in five categories: hand tools, power tools,
garden tools, cleaning tools, and outdoor gear.

To borrow, create a reservation. Six states: requested, approved,
picked up, returned, denied, and cancelled. A PostgreSQL
exclusion constraint prevents double-booking at the database
level. One-day rentals are supported.

Supporting tables handle invites, email tokens, notifications,
reviews, damage reports, and the admin audit log.

Three design choices: UUIDs so IDs can't be guessed.
PostgreSQL enums so bad states can't sneak in. And soft-delete
with PII anonymization — the row stays for past records,
but email and name are scrubbed.

That is the domain model.

Q9: What is automatic rotation?
================================

It is our two-token JWT system. Login gives you two tokens:
an access token (15 minutes) sent with every API request,
and a refresh token (7 days) used only to get a new pair.
When the access token expires, the frontend automatically
calls the refresh endpoint. The server validates the refresh
token, then issues a brand new access token AND a brand new
refresh token — the old pair is discarded. Each refresh
produces fresh credentials, the user never sees it, and a
stolen token has a limited window of usefulness.

Q10: What is PII security?
===========================

PII stands for Personally Identifiable Information — data
that can identify a person. In our app: email, full name,
neighborhood, and profile photo.

When a member deletes their account, we do not hard-delete
the row because it would break foreign keys. Instead,
_anonymize_user() scrubs the PII: email becomes
deleted+<uuid>@example.com, password hash is cleared,
and the name becomes "Deleted User". The row stays,
the person is gone.

There is also an active-reservation guard: you cannot delete
if you have active borrows or tools out on loan.
